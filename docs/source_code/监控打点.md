# 监控打点

feapder 内置打点监控，将爬虫运行指标写入 InfluxDB 1.x，再由 Grafana 看板可视化。整套监控只需配置 InfluxDB 地址、在 Grafana 导入看板 JSON 即可使用，无需依赖其他平台。

监控数据默认保留 180 天，滚动删除。

## 配置 InfluxDB

在项目 `setting.py` 或环境变量中配置连接（无认证时账号密码留空即可）：

```python
INFLUXDB_HOST = "106.54.14.129"
INFLUXDB_PORT = 8086
INFLUXDB_DATABASE = "crawler"
INFLUXDB_USER = ""
INFLUXDB_PASSWORD = ""
# 可选：自定义 measurement 名称，默认 crawler
INFLUXDB_MEASUREMENT = "crawler"
# 可选：站点名，默认取项目目录名，用于看板按站点筛选
PROJECT_NAME = "taobao"
```

数据库与保留策略会在爬虫首次运行时自动创建。

## 数据模型

所有打点写入同一个 measurement（默认 `crawler`），通过 tag 区分维度：

- `project`：站点/项目名，默认取项目目录名
- `spider`：爬虫名
- `_classify`：指标分组，`document` 为下载情况，`<表名>` 为入库情况，其余为自定义
- `_key`：序列名（折线名）
- `_type`：`counter` / `timer` / `store`

字段：`_count`（计数）、`_duration`（耗时，秒）、`_value`（快照值）。

## 内置指标

- 使用 `yield item` 入库的数据，`_classify` 为表名，折线展示每个字段是否有值以及数据总量（`total count`）。
- `document` 为下载情况，包含 `download_total` / `download_success` / `download_exception` / `parser_exception`。
- `http_status` 为 HTTP 状态码分布，`_key` 为具体状态码（200/404/500...），在收到响应时记录（早于 `validate`，被 `validate` 拒绝的 4xx/5xx 也会统计）。
- `latency` 为请求响应耗时，`_key` 为爬虫名，`_duration` 为耗时秒数，官方 Grafana 看板展示 P95 曲线。
- `runtime` 为运行状态，爬虫启动后由后台线程按 `METRICS_RUNTIME_INTERVAL`（默认 10 秒）周期上报 `store` 点：`heartbeat`（Unix 秒级心跳时间戳）、`memory`（进程内存占用，MB）。这些点额外带 `hostname` 与 `pid` tag 以区分进程实例，其中 `hostname` 取本机内网 IP（探测失败时回退为机器名），分布式或多进程运行同一爬虫时，看板按实例展示，可快速定位异常进程。看板按 1 分钟心跳桶展示实例在线状态，并展示内存曲线，其中内存采集依赖 `psutil`（已随 feapder 一并安装）。

若想监控其他指标，参考下面的自定义监控。

## 自定义监控

举例：编写 `test_metrics.py` 代码如下：

```python
from feapder.utils import metrics

# 初始化打点系统
metrics.init()

metrics.emit_counter("key", count=1, classify="test")

metrics.close()
```

再来解释下：

```python
metrics.emit_counter("key", count=1, classify="test")
```

- key 对应折线
- count 对应点数
- classify 对应图表分组（看板分类）

若代码如下：

```python
from feapder.utils import metrics

# 初始化打点系统
metrics.init()

metrics.emit_counter("key", count=1, classify="test")
metrics.emit_counter("key2", count=1, classify="test")
metrics.emit_counter("key3", count=1, classify="test")

metrics.emit_counter("哈哈", count=1, classify="test2")

metrics.close()
```

将生成两个分组：`test` 含 3 条折线，`test2` 含 1 条折线。

如在 feapder 爬虫中使用，示例如下：

```python
import feapder
from feapder.utils import metrics


class TestSpider(feapder.AirSpider):
    def start_requests(self):
        yield feapder.Request("https://www.baidu.com")

    def parse(self, request, response):
        # 自定义监控
        metrics.emit_counter("success", count=1, classify="自定义的监控指标")


if __name__ == "__main__":
    TestSpider().start()
```

在 feapder 爬虫中只需导包，然后调用 `metrics.emit_counter` 即可，不需要关心 `metrics.init` 和 `metrics.close`；打点会自动带上当前 `project` 与 `spider` tag。若在 scrapy 或其他 python 脚本中使用，必须自行调用 `metrics.init` 和 `metrics.close`。

## 使用限制

监控采用模块级单例（全局唯一 `_emitter` 与一份默认 tag），同一进程内只支持运行单个爬虫，不支持在同一进程内并发运行多个不同爬虫，否则指标的 `project` / `spider` tag 会归属混乱。多个爬虫请各自独立进程运行。同一爬虫在同进程内顺序重复运行（如循环 `start()`）不受影响。

## Grafana 可视化

在 Grafana 中新增 InfluxDB 数据源（InfluxQL，指向上面配置的地址与库），导入 `monitoring/grafana/dashboards/feapder-crawler.json` 看板，即可按 站点 -> 爬虫 -> 时间 维度查看监控。详细步骤见 `monitoring/grafana/README.md`。
